- **Onboarding an EXISTING repo now stages `.githooks/*` at mode `100755` (staging only — never a commit on user history).** The #317 fix staged the exec bit only inside `initGitAndCommit`'s bootstrap commit, which `check()` skips for any target with prior commits — the path every real onboarding (civ-sim, lifeloot) took — so Windows hosts (`core.filemode=false`) still ended up committing hooks at `100644` and Unix clones silently skipped the guards. `writeGithooks.apply()` now stages `git update-index --add --chmod=+x` via the same shared helper whenever the target is already a git repo (index mode is sticky, so the agent's later plain `git add` + commit preserves `100755`); `check()` reports `needs-apply` for a hook tracked at `100644` so re-onboard/update runs repair the drift; `verify()` now inspects the staged mode too, making it honest on win32.
