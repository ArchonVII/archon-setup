- **Fresh onboards now COMMIT `.githooks/*` at mode `100755`.** `writeGithooks` chmodded hooks on disk only; on Windows (`core.filemode=false`) the bootstrap staging step recorded them as `100644`, so a Unix clone of a Windows-onboarded repo got non-executable commit-msg/pre-commit guards that git silently skips (the end-to-end remainder of #294/#314). `initGitAndCommit` now stages the executable bit explicitly (`git update-index --add --chmod=+x`) for every present hook entrypoint before the bootstrap commit; regression test runs a real onboard and asserts the committed tree modes.
